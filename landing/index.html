Get Started See it in action
pipeline.ts
import { Pipeline } from 'pixflow'

// 100 images. GPU-processed. Under 3 seconds.

const pipe = Pipeline.create()
  .orient()
  .resize({ width: 800, fit: 'contain' })
  .brightness(0.05)
  .unsharpMask({ amount: 0.3, radius: 1.0 })
  .encode({ format: 'webp', quality: 0.85 })

const results = await pipe.batch(files, {
  concurrency: 4,
  onProgress: (done, total) => updateUI(done, total)
})

// results[i] → { blob, width, height, stats }
Why pixflow

Privacy is architecture

Photos never leave the user's device. No server uploads, no logs, no cloud storage. GDPR compliance built in, not bolted on.

GPU-native speed

WebGPU compute shaders process pixels in parallel. 100 images × 2K resized in under 3 seconds on an M1 Mac. Canvas2D can't compete.

Declarative pipelines

Chain filters with a fluent builder API. No imperative canvas calls, no manual texture management. Read the code, understand the pipeline.

Zero cost at scale

Your users' GPUs do the work, not your servers. Bandwidth drops 90%. Server-side image processing becomes optional. Your AWS bill shrinks.

Each filter runs as a WGSL compute shader on the GPU. Intermediate textures use a ping-pong pattern — N filters only need 2 textures. Pipeline cache eliminates redundant shader compilation across batch items.
